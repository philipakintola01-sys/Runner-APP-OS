You are a senior mobile architect, product designer, UX researcher, endurance athlete, data analyst, and full-stack engineer.

Your task is to design and build a production-quality mobile application called RunOS.

Do not build a demo.
Do not build placeholder screens.
Do not generate fake functionality.
Do not use dummy data except where explicitly required during development.

Think deeply before implementation.

=================================================
PRODUCT VISION
=================================================

RunOS is an offline-first running operating system designed for students, amateur runners, endurance athletes, and marathon enthusiasts.

The app should help runners:

1. Track runs accurately.
2. Understand performance trends.
3. Compare current performance against previous runs.
4. Maintain consistency through streaks and goals.
5. Generate beautiful shareable run cards for social media.
6. Operate reliably even without internet access.

The philosophy is:

"Track. Understand. Improve. Share."

The experience should feel focused, modern, data-rich, and athlete-centric.

=================================================
TARGET USER
=================================================

Primary User:

- Student
- Runs frequently
- Trains indoors and outdoors
- Wants meaningful analytics
- Wants visual progress
- Wants to share achievements
- Often has inconsistent internet access

The app should prioritize simplicity over feature bloat.

=================================================
CORE PRINCIPLES
=================================================

1. Offline-first.

The app must continue functioning without internet.

Users should always be able to:

- start run
- stop run
- view history
- view analytics
- generate run cards

while offline.

Data syncs automatically when internet becomes available.

-------------------------------------------------

2. Fast launch.

User should be able to begin a run within 2 taps.

-------------------------------------------------

3. Performance over aesthetics.

Accurate tracking is more important than visual effects.

-------------------------------------------------

4. Data should produce insight.

Never show numbers without interpretation.

Bad:

"You ran 5km."

Good:

"You ran 5km 8% faster than your average pace over the last 5 runs."
